Effects

Delay, distortion, and reverb — the three fundamental effect categories. Each demonstrates a different signal routing pattern in the Web Audio node graph.

Test source

Use a melodic arpeggio or a single plucked note to hear how each effect treats the signal. All three effects run in series in the chain below.

Delay

DelayNode

Stores audio in a circular buffer and replays it after a set time. Feeding the output back into the input creates repeating echoes.

375 ms
45 %
60 %
Dry/wet routing. The source connects to both a dry GainNode and the effect input. The effect output connects to a wet GainNode. Both gains feed the master output. Adjusting the two gain values blends between unprocessed (dry) and effected (wet) signals.

Distortion

WaveShaperNode

Applies a non-linear transfer curve to the waveform — soft clipping adds harmonics (warmth), hard clipping adds more (fuzz/grunge). The curve canvas shows the waveshaper function.

50
60 %
Transfer curve. WaveShaperNode maps each input sample value through a curve array. A value of −1 maps to curve[0], +1 maps to curve[curve.length-1], and intermediate values are interpolated. A straight diagonal is unity gain; bending it introduces distortion.

Reverb

ConvolverNode

True convolution reverb applies a recorded room impulse response (IR) to the signal. Here we synthesise the IR procedurally — decaying noise burst — which sounds like a small room or spring reverb.

1.5 s
2.0
50 %
Impulse response synthesis. A real IR is a recording of a clap or starter pistol in a room. Synthesised IRs fill an AudioBuffer with white noise multiplied by an exponential decay envelope — noise × e^(−t × decay). The diffusion parameter controls how quickly the early reflections spread into a dense tail.
